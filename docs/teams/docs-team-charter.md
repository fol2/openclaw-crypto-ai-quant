# Documentation Team Charter

## Mission

Keep the Rust runtime migration understandable, reviewable, and safe to hand
off across agent and human contributors.

## Standing Deliverables

- ADR updates for architectural decisions
- migration programme status and phase boundaries
- runbook diffs for service and operator changes
- parity matrix updates as new runtime modes converge
- legacy deletion ledger updates before any module removal

## Working Agreement

- documentation ships in the same PR as the behaviour it explains
- documentation must describe current truth, not aspirational intent
- public-facing and code-adjacent prose uses UK English
- deletions are never documented after the fact; they are recorded before merge
